## [0.5.7] ŌĆö 2026-06-12 ŌĆö SCIM hardening, RISC fixes & discovery

### Fixed
- **RISC lifecycle SET payloads** ŌĆö `account-purged`, `account-disabled`, and `account-enabled` no longer inject a `subject` object into the per-event `events` body; user identity stays in the top-level `sub_id` claim per SSF 1.0 ┬¦5.1 (strict receivers such as Apple reject schema-invalid RISC bodies)
- **Apple SCIM 409 recovery** ŌĆö username-based recovery after `409 Conflict` only when the matched Apple user has no `externalId` or the same `externalId` as Authentik (prevents overwriting another user's managed Apple ID)
- **Apple SCIM group filter** ŌĆö group membership fetched via Authentik `GET /api/v3/core/groups/{pk}/` instead of a non-existent users sub-resource
- **Apple SCIM updates** ŌĆö user changes sent with `PATCH` instead of `PUT`
- **Webhook email extraction** ŌĆö whitespace-only or non-string `email` values treated as missing
- **404 exception handler** ŌĆö route-level `HTTPException` responses preserve `detail` and `headers` (e.g. `WWW-Authenticate` on 401); custom discovery hint only for unmatched routes

### Added
- **`GET /`** ŌĆö minimal public discovery (service name, `APP_VERSION`, link to `/.well-known/ssf-configuration`); HTML or JSON via `Accept`
- **Custom 404 responses** ŌĆö unknown paths return JSON or HTML with a hint to the SSF well-known URL instead of a bare FastAPI error
- **`SSF_ENABLE_OPENAPI`** ŌĆö gates `/docs`, `/redoc`, and `/openapi.json` (default `false`)

### Changed
- FastAPI `version` reads `APP_VERSION` (set at image build time; `dev` locally)
- SCIM sync logs use structured fields instead of empty `{}` placeholders
- Local dev, CI, and Docker image aligned on **Python 3.14** (`.python-version`, GitHub Actions, `python:3.14-slim-bookworm`)

### Dependencies
- `PyJWT[crypto]` `>=2.13.0`, `cryptography` `>=48.0.1`
- `ruff` `>=0.15.17`, `pip-audit` `>=2.10.1`, `deptry` `>=0.25.1` (dev)
- Docker base image `python:3.14-slim-bookworm` digest bump
